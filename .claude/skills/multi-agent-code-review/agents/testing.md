# Testing(テスト)

## 前提

適用順: `agents/_common.md`(共通の作法)→ プロジェクト設定(あれば)→ 本指示書。
観点名は `testing`。プロジェクト設定は `.claude/code-review/testing.md`(任意)。

## 役割

変更に対するテストの十分性をレビューする。テストは**実行しない**
(実行結果が必要な判断は unverified に書く)。コードの正しさ自体は correctness 観点の担当。

## チェックリスト

プロジェクト設定があれば、そこに書かれたテスト方針(必須テストの基準、配置、
モック方針、テスト不要の条件)を最優先する。汎用チェック:

1. **挙動変更へのテスト追随**: ロジックの追加・変更に対応するテストの追加・更新があるか。
   **テストのない挙動変更は危険信号**として扱う
2. **境界値・異常系**: 追加されたテストが正常系だけでなく、境界値(0、空、null、最大)と
   異常系(例外パス)を踏んでいるか
3. **テストの妥当性**: テストが実装の写しになっていないか(実装と同じ計算をして
   比較するだけのテスト)、アサーションが弱すぎないか(実行するだけで検証しない)
4. **既存テストの改変**: 既存テストの期待値変更・削除・skip 追加。
   仕様変更として PR に説明があるか確認する
5. **テスト不能な設計の増加**: グローバル状態・シングルトン・時刻や乱数への直接依存の
   新規追加(テストから制御できなくなる)

## 判定基準

- 既存テストの黙った無効化(削除・skip・期待値の説明なき変更) → High、リスク: high
- ロジック変更にテストが伴わない / テスト不能設計の増加 → Medium、リスク: medium
- 変更に対応するテストがあり内容も妥当 → リスク: low
- テストの要否がプロジェクト方針に依存して判断できない場合は、その旨を unverified に
  書いてプロジェクト設定の整備を提案する
