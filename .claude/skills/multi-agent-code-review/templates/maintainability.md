## 命名規則

<!-- 言語標準の命名規則(camelCase 等)はリンターに任せ、
プロジェクト固有の語彙・接尾辞などを書く。 -->

- (例) ユースケースクラスは `<動詞><対象>UseCase` と命名する(`FetchUserUseCase`)
- (例) 「プレイヤー」は `Player` に統一する(`User` や `Character` と混在させない)

## エラーハンドリング

- (例) 例外を握りつぶさない。回復不能なエラーは上位に伝播させる
- (例) ユーザー向けエラーメッセージは `ErrorMessages` 定数クラスに集約する

## ログ

- (例) `print` / `Console.WriteLine` 直接呼び出し禁止。`Logger` を使う
- (例) 個人情報・トークンをログに出力しない

## パフォーマンス制約

<!-- このプロジェクトでの「ホットパス」がどこかを明示する。
maintainability 観点はホットパスでのアンチパターンを重点的に見る。 -->

- (例) `Update()` / 毎フレーム実行される経路では新規アロケーションを追加しない
- (例) API ハンドラ内でのループ内クエリ発行(N+1)を禁止する

## マジックナンバー方針

- (例) ゲームの調整値(速度・間隔・閾値)は ScriptableObject に定義し、ハードコード禁止
- (例) 2 回以上使う文字列キーは定数化する
