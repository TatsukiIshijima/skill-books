## 役割

シリアライズドアセット(Scene / Prefab / ScriptableObject / .meta)の変更健全性をレビューする。

## 起動条件

<!-- この観点をどんな diff のときに起動するか。パスパターンで書く。
どのパターンにも該当しない diff ではこの観点は起動されない(トークン節約)。 -->

変更ファイルに以下のいずれかが含まれるとき:

- `**/*.unity` / `**/*.prefab` / `**/*.asset` / `**/*.meta`
- `ProjectSettings/**`

## チェックリスト

1. **YAML diff を行単位で読解しない**: Scene / Prefab の diff は巨大かつ難読。
   変更されたアセットの一覧・件数と、読み取れる範囲の変更種別(参照変更・
   コンポーネント追加削除・値変更)のみを把握し、詳細が読み取れない場合は
   unverified として扱う(分かったふりをしない)
2. **meta 整合性**: 新規ファイルに対応する .meta の追加漏れ、削除ファイルの .meta 残り
3. **コード変更との随伴性**: PR の目的(コード変更)と無関係な Scene / Prefab 変更が
   紛れ込んでいないか
4. **ScriptableObject のパラメータ変更**: 変更が PR の説明と整合するか

## 判定基準

- .meta の不整合(追加漏れ・孤児) → High(マージ前修正必須)
- PR の説明にない Scene / ProjectSettings の変更 → Medium 以上
- 内容を確認できない大きなアセット変更 → Medium(人間による Unity エディタ上での確認を提案)
- ScriptableObject のパラメータ調整のみで説明と整合 → Low(指摘なしでよい)
