# 共通ルール(言語・プロジェクト非依存)

すべてのPRに適用する。パスパターンはリポジトリルートからのglob。

## 人間レビュー必要パターン

1つでも該当したら 🔴 で即決し、Stage 2 をスキップする。

| ルール名 | パターン / 条件 | 理由 |
|---|---|---|
| DBマイグレーション | `**/migrations/**`, `**/migrate/**`, `**/*migration*.sql`, `**/alembic/**`, `**/flyway/**` | 不可逆な変更になりやすく、ロールバック手順の確認が必要 |
| 認証・認可・決済・暗号 | パスまたはdiff内容に auth / oauth / session / permission / payment / billing / crypto / secret / token 関連の変更 | セキュリティ影響が大きく、微妙なミスが重大事故につながる |
| CI/CD・インフラ | `.github/workflows/**`, `**/Dockerfile*`, `**/docker-compose*`, `**/terraform/**`, `**/*.tf`, `**/k8s/**`, `**/helm/**` | ビルド・デプロイ経路の変更は全体に波及し、秘匿情報の漏洩リスクもある |
| 依存関係マニフェスト | `**/package.json`, `**/build.gradle{,.kts}`, `**/pom.xml`, `**/requirements*.txt`, `**/pyproject.toml`, `**/Gemfile`, `**/go.mod`, `**/Cargo.toml`, `**/Podfile`, `**/*.csproj` の依存追加・更新(lockfileのみの再生成は除く) | サプライチェーンリスクとライセンス確認が必要 |
| 公開API・スキーマ | `**/openapi*`, `**/swagger*`, `**/*.proto`, `**/graphql/**/*.graphql`, `**/schema/**` | 外部との契約変更は後方互換性の確認が必要 |
| 変更規模が大きい | 変更 500行超 または 20ファイル超(自動生成ファイル・lockfileを除いて数える) | 大きなPRは機械判定の見落としリスクが高く、人間の全体把握が必要 |

## 人間レビュー不要候補パターン

変更ファイルの**すべて**が以下のいずれかに該当する場合のみ 🟢 の候補とする。
diffを確認し、パターンの想定どおりの内容であることを確かめてから確定すること。

| ルール名 | パターン / 条件 | 確認ポイント |
|---|---|---|
| ドキュメントのみ | `**/*.md`, `**/docs/**`, `LICENSE`, `**/*.txt` | コードブロック内にコピペされる想定のコマンド等が誤っていないか |
| コメント・typoのみ | diffがコメント行・文字列リテラル内のtypo修正のみ | 文字列がユーザー表示やAPIレスポンスに使われる場合は挙動変更になる点に注意 |
| lockfileのみ | `**/package-lock.json`, `**/yarn.lock`, `**/pnpm-lock.yaml`, `**/Gemfile.lock`, `**/go.sum`, `**/Cargo.lock`, `**/Podfile.lock` のみの再生成 | マニフェスト側の変更を伴う場合は「依存関係マニフェスト」ルールが優先 |
| 自動整形のみ | フォーマッタ適用によるインデント・改行・import順のみの変更 | ロジック行の変更が紛れていないか |
