# Unity固有ルール

Unityプロジェクト(`*.unity` / `*.meta` / `Assets/` / `ProjectSettings/` を含むPR)に適用する。
共通ルール(`../common/rules.md`)に上乗せして評価する。

## 人間レビュー必要パターン

| ルール名 | パターン / 条件 | 理由 |
|---|---|---|
| プロジェクト設定 | `ProjectSettings/**`, `Packages/manifest.json` | ビルド設定・入力・物理設定などプロジェクト全体に波及する |
| シェーダー・レンダリング | `**/*.shader`, `**/*.shadergraph`, `**/*.hlsl`, `**/*.cginc`, RenderPipeline アセット | 見た目とパフォーマンスへの影響が実行してみないと分からない |
| Addressables / アセットバンドル | `**/AddressableAssetsData/**`, アセットバンドル設定 | 配信・ロード経路の変更はリリース後に発覚しやすい |
| シーン・Prefabの大きな変更 | `**/*.unity`, `**/*.prefab` の大量差分(マージ事故が起きやすい) | YAMLの機械的diffでは意図が読み取れず、Unity上での目視確認が必要 |
| ネイティブプラグイン | `Assets/Plugins/**` のバイナリ・ネイティブコード変更 | プラットフォーム別の動作確認が必要 |

## 人間レビュー不要候補パターン

| ルール名 | パターン / 条件 | 確認ポイント |
|---|---|---|
| .metaのみ | `**/*.meta` のみの変更(GUID再生成・importer設定なし) | GUIDが変わっている場合は参照切れの恐れがあるため不要候補から外す |
| アセットの軽微な再インポート | importerバージョン更新による機械的な差分のみ | 設定値(圧縮・最大サイズ等)の変更が混ざっていないか |
